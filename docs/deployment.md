# Deployment Instructions

## Overview

EduSpark is a static PWA that can be deployed to any static hosting service. The build output is in the `dist/` directory.

## Build for Production

```bash
npm run build
```

This produces:
- `dist/index.html` - Entry point
- `dist/assets/` - CSS and JS bundles
- `dist/sw.js` - Service worker
- `dist/workbox-*.js` - Workbox runtime
- `dist/manifest.webmanifest` - PWA manifest
- `dist/icons/` - App icons

## Deployment Options

### Vercel (Recommended)

1. Install Vercel CLI:
   ```bash
   npm i -g vercel
   ```

2. Deploy:
   ```bash
   cd edu-spark
   vercel
   ```

3. Configure environment variables in Vercel dashboard:
   - `VITE_APPWRITE_ENDPOINT`
   - `VITE_APPWRITE_PROJECT_ID`
   - `VITE_APPWRITE_DATABASE_ID`
   - `VITE_APPWRITE_FN_*`

4. For subsequent deployments:
   ```bash
   vercel --prod
   ```

### Netlify

1. Install Netlify CLI:
   ```bash
   npm i -g netlify-cli
   ```

2. Deploy:
   ```bash
   cd edu-spark
   netlify deploy --prod --dir=dist
   ```

3. Configure environment variables in Netlify dashboard.

### Cloudflare Pages

1. Push code to GitHub/GitLab
2. Connect repository in Cloudflare Pages dashboard
3. Set build command: `npm run build`
4. Set build output directory: `dist`
5. Configure environment variables

### GitHub Pages

1. Add to `vite.config.ts`:
   ```typescript
   export default defineConfig({
     base: '/edu-spark/',
     // ... other config
   });
   ```

2. Build and deploy:
   ```bash
   npm run build
   # Push dist/ to gh-pages branch
   ```

## PWA Requirements

### HTTPS Required

PWAs require HTTPS in production. All recommended hosting options provide HTTPS automatically.

### Service Worker

The service worker is generated by Workbox during build. It:
- Caches the app shell (HTML, CSS, JS)
- Caches downloaded readings
- Provides offline fallback

### Manifest

The PWA manifest (`manifest.webmanifest`) enables:
- "Add to Home Screen" on mobile
- Standalone display mode
- App icons
- Theme color

## Environment Variables

Set these in your hosting platform's environment variables:

| Variable | Required | Description |
|----------|----------|-------------|
| `VITE_APPWRITE_ENDPOINT` | Yes | Appwrite API endpoint |
| `VITE_APPWRITE_PROJECT_ID` | Yes | Appwrite project ID |
| `VITE_APPWRITE_DATABASE_ID` | Yes | Database ID |
| `VITE_APPWRITE_FN_GET_QUESTIONS` | Yes | Function ID |
| `VITE_APPWRITE_FN_SUBMIT_QUESTION` | Yes | Function ID |
| `VITE_APPWRITE_FN_TOGGLE_VOTE` | Yes | Function ID |
| `VITE_APPWRITE_FN_TEACHER_PROGRESS` | Yes | Function ID |

## Post-Deployment Checklist

1. **Test PWA installation** - Visit on mobile, tap "Add to Home Screen"
2. **Test offline mode** - Load app, go offline, verify it works
3. **Test service worker** - Check DevTools → Application → Service Workers
4. **Test manifest** - Check DevTools → Application → Manifest
5. **Verify environment variables** - Ensure Appwrite connection works
6. **Test authentication** - Register and login
7. **Test sync** - Create content, verify it syncs to Appwrite

## Custom Domain

### Vercel
1. Go to project settings → Domains
2. Add your custom domain
3. Configure DNS records as shown

### Netlify
1. Go to site settings → Domain management
2. Add custom domain
3. Configure DNS records

## Monitoring

### Appwrite Console
- Monitor API usage
- Check function executions
- View error logs

### Browser DevTools
- Application tab: Check service worker, manifest, storage
- Network tab: Verify API calls
- Console: Check for errors

## Troubleshooting

### Service Worker Not Updating
- Hard refresh (Ctrl+Shift+R)
- Clear site data in DevTools
- Check service worker status in DevTools

### PWA Not Installing
- Verify HTTPS
- Check manifest is valid
- Check service worker is registered
- Verify icons exist

### Environment Variables Not Working
- Ensure variables start with `VITE_`
- Rebuild after changing variables
- Check hosting platform logs
