# Learning is Fun

Offline-first educational flashcard and reading Progressive Web App.

**Production URL:** https://zumboggo.github.io/Learningisfun/

## Quick Start

### Prerequisites

- Node.js 20+
- npm 9+

### Installation

```bash
git clone https://github.com/zumboggo/Learningisfun.git
cd Learningisfun
npm install
```

### Local Development

```bash
# Start dev server (http://localhost:5173/Learningisfun/)
npm run dev

# Run TypeScript type checking
npx tsc -b

# Run tests
npm test

# Build for production
npm run build

# Preview production build locally
npm run preview
```

### Environment Variables

Copy `.env.example` to `.env` for future Appwrite configuration:

```bash
cp .env.example .env
```

No environment variables are required for the base PWA to function.

## Tech Stack

| Layer | Technology |
|-------|-----------|
| Framework | React 19, TypeScript |
| Build | Vite 8 |
| Styling | TailwindCSS 4 |
| Routing | React Router 7 (HashRouter) |
| Local DB | Dexie (IndexedDB) |
| Backend | Appwrite Cloud (when configured) |
| PWA | vite-plugin-pwa + Workbox |
| Testing | Vitest, Testing Library |

## PWA Features

- **Installable** — Add to home screen on Android, iOS, and desktop
- **Offline-ready** — Application shell cached via service worker
- **Auto-update** — Prompts user when a new version is available
- **Standalone display** — Runs without browser chrome when installed

### Testing PWA Installation

1. Deploy to GitHub Pages (see below)
2. Open the production URL in Chrome
3. Click the install icon in the address bar, or use the three-dot menu → "Install app"
4. On mobile (Android), tap "Add to Home Screen" from the browser menu

### Testing Offline Mode

1. Open the production URL and let it fully load
2. Open DevTools → Application → Service Workers (verify SW is active)
3. Open DevTools → Network → check "Offline"
4. Reload the page — it should still load from cache

## GitHub Pages Deployment

Deployment is automatic via GitHub Actions when pushing to the `master` branch.

### Workflow

1. Push to `master`
2. GitHub Actions runs: `npm ci` → TypeScript check → tests → `npm run build`
3. The `dist` folder is deployed to GitHub Pages

### Manual Setup (one-time)

1. Go to your repository → **Settings** → **Pages**
2. Under **Source**, select **GitHub Actions**
3. The workflow will deploy automatically on the next push

### Build Configuration

- **Base path:** `/Learningisfun/` (configured in `vite.config.ts`)
- **Router:** `HashRouter` — all routes use `#` (e.g., `/#/dashboard`)
- **Service worker:** Auto-generated by vite-plugin-pwa with Workbox

## Architecture

### Offline-First Design

Every write goes to IndexedDB first, then queues for cloud sync:

1. User action → Write to IndexedDB
2. Add operation to sync queue
3. Sync engine processes queue when online
4. UI updates via Dexie `liveQuery` reactivity

### Sync Triggers

- App opens
- Internet connection restored
- Tab becomes visible
- After local change (debounced 2s)
- Manual sync button
